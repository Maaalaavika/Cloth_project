Full Prompt Text
You are building Prompt 5 of a Clothing Exchange & Swap Marketplace (MERN Stack + Socket.io). The project already has: user auth, item listing, swap request flow, and profile pages (Prompts 1–4). ─── PART A: IN-APP REAL-TIME CHAT ─────────────────────────────── Goal: Each swap request has its own private chat thread between the two users involved. Backend (Express + Socket.io): 1. Create a Chat model in MongoDB: - Fields: swapRequestId (ref SwapRequest), messages: [{ sender (ref User), text, imageUrl, readBy: [userId], createdAt }] 2. Socket.io server setup (server.js): - On connection: authenticate user via JWT from socket handshake query - joinRoom(swapRequestId): join a socket room named after the swapRequestId - sendMessage(swapRequestId, text, imageUrl): save message to DB, emit to room - markRead(swapRequestId): update readBy for all messages not read by this user 3. REST API routes (chatRoutes.js): - GET /api/chat/:swapRequestId — fetch full chat history (auth required, only swap participants) Frontend (React): 1. ChatWindow.jsx component: - Props: swapRequestId, currentUserId, otherUser - Connects to socket on mount, joins the room, disconnects on unmount - Displays scrollable message list: right-aligned (sender), left-aligned (receiver) - Show read receipt: "Seen" under last sent message if other user has readBy entry - Image sharing: small file input, convert to base64, send as imageUrl - Auto-scroll to latest message 2. NavBar unread badge: - On socket connect, listen for unreadCount event (total unread messages across all chats) - Show red badge number on the chat icon in navbar; clear on chat open ─── PART B: NOTIFICATIONS SYSTEM ──────────────────────────────── Goal: Notify users of key swap lifecycle events, both in-app and via email. Trigger these 4 notification types: • NEW_SWAP_REQUEST — someone sent you a swap request • REQUEST_ACCEPTED — your swap request was accepted • REQUEST_DECLINED — your swap request was declined • NEW_MESSAGE — you received a new chat message (debounce: only if chat not open) • SWAP_COMPLETED — both parties confirmed; swap marked done Backend: 1. Notification model (MongoDB): - Fields: recipient (ref User), type (enum above), message (string), relatedSwap (ref SwapRequest), isRead (Boolean, default false), createdAt 2. createNotification(userId, type, message, swapId) utility function — saves to DB, emits socket event "newNotification" to that user's personal room (room = userId) 3. Call createNotification() in the right places: - swapController.js: on create → notify item owner; on accept/decline → notify requester; on complete → notify both - chatController.js (socket): on sendMessage → notify recipient if not in chat room 4. REST API: - GET /api/notifications — get all notifications for logged-in user (sorted newest first) - PATCH /api/notifications/:id/read — mark one as read - PATCH /api/notifications/read-all — mark all as read 5. Nodemailer (optional, toggle via .env EMAIL_NOTIFICATIONS=true): - Send a plain-text email on NEW_SWAP_REQUEST and SWAP_COMPLETED - Use a transporter configured from SMTP env vars (EMAIL_HOST, EMAIL_PORT, EMAIL_USER, EMAIL_PASS) Frontend (React): 1. NotificationBell.jsx (navbar): - Bell icon with badge showing unread count - On click: dropdown panel showing last 10 notifications - Each item: icon by type, message text, timestamp (time-ago format), gray if read - Click a notification → mark as read + navigate to relevant swap page - "Mark all read" button at top of dropdown 2. Socket listener in a useNotifications() custom hook: - On mount: fetch notifications via GET /api/notifications - Listen for "newNotification" socket event → prepend to list, increment badge - Expose { notifications, unreadCount, markRead, markAllRead } ─── DELIVERABLES ───────────────────────────────────────────────── Provide complete code for: Backend: models/Chat.js, models/Notification.js, utils/createNotification.js, routes/chatRoutes.js, routes/notificationRoutes.js, socket/chatSocket.js (Socket.io event handlers), and relevant additions to server.js Frontend: components/ChatWindow.jsx, components/NotificationBell.jsx, hooks/useNotifications.js, and navbar badge integration snippet Follow these constraints: - Auth middleware: protect all REST routes with existing verifyToken middleware - Socket auth: verify JWT on handshake; reject unauthenticated connections - Only swap participants may access a chat thread (enforce server-side) - Use existing User, SwapRequest, Item Mongoose models from earlier prompts - Tailwind CSS for all new components (consistent with existing UI) - Include socket cleanup (disconnect, off) in all React useEffect returns
What this prompt covers
  • Chat model — MongoDB schema with messages, readBy, imageUrl fields
  • Socket.io server — JWT auth on handshake, swap rooms, sendMessage, markRead events
  • ChatWindow.jsx — real-time messages, read receipts, image sharing, auto-scroll
  • Unread badge — navbar chat icon with live unread count from socket
  • Notification model — 5 trigger types stored in MongoDB
  • createNotification utility — called from swap & chat controllers
  • NotificationBell.jsx — dropdown with mark-read, time-ago, type icons
  • useNotifications() hook — socket listener + REST fetch combined
  • Nodemailer — optional email on new swap request & swap completion
  • Security — participant-only chat access enforced server-side
