Full Prompt Text
You are building Prompt 6 of a Clothing Exchange & Swap Marketplace (MERN + React + Tailwind CSS). The project already has: user auth, item listing with image upload, swap request flow, real-time chat, and a notifications system (Prompts 1–5). ─── PAGE 1: USER DASHBOARD ────────────────────────────────────── Route: /dashboard (protected, requires auth) A. Stat Cards Row (top of page) Fetch from GET /api/users/me/stats → { totalListed, successfulSwaps, trustScore } Render three summary cards side by side (stack on mobile): • Total Items Listed — icon: ti-shirt • Successful Swaps — icon: ti-arrows-exchange • Trust Score — shown as "X / 5.0 ⭐", colored green if ≥4, amber if 2–3.9, red if <2 B. Tab Navigation (My Listings | Swap Requests | Saved Items | Swap History) Use a sticky tab bar below the stat cards. Active tab has a green underline indicator. Persist active tab in URL query param: /dashboard?tab=requests ── Tab 1: My Listings ── Fetch: GET /api/items?owner=me Sub-tabs (pill style): All | Active | Swapped | Draft Each listing shown as a compact horizontal card: - Thumbnail (60x60px, object-cover, rounded-lg) - Title, category badge, condition chip - Status chip: Active (green), Swapped (purple), Draft (gray) - Action buttons: Edit (navigates to /items/:id/edit), Delete (confirm modal), Relist if Swapped Empty state: illustration placeholder + "List your first item" CTA button ── Tab 2: Swap Requests ── Fetch: GET /api/swaps?type=incoming and GET /api/swaps?type=outgoing (parallel) Two collapsible sections: "Incoming Requests" and "My Outgoing Requests" Each swap card shows: - Other user's avatar + name + location - Their offered item thumbnail + title - Your item thumbnail + title (the one they want) - Status chip: Pending (amber), Accepted (green), Declined (red), Completed (blue) - For incoming + Pending: Accept and Decline buttons → call PATCH /api/swaps/:id/status - For any status: "Open Chat" button → navigate to /chat/:swapId Empty state per section with contextual message ── Tab 3: Saved Items ── Fetch: GET /api/users/me/saved Masonry-style grid (2 cols mobile, 3 cols desktop) of saved item cards: - Item image, title, owner avatar + name, condition badge - "Request Swap" button → navigates to item detail page - Unsave button (heart icon, filled) → DELETE /api/users/me/saved/:itemId, optimistic UI remove Empty state: "Browse listings and save items you love" ── Tab 4: Swap History ── Fetch: GET /api/swaps?status=completed Chronological list of completed swaps: - Swap date, both items exchanged (thumbnail + title each side) - Partner avatar + name - Review prompt: if no review given yet → show "Leave a Review" button → opens inline review form: star rating (1–5) + text comment → POST /api/reviews - If review already given → show the submitted stars + comment (read-only) Empty state: "Your completed swaps will appear here" ─── PAGE 2: HOMEPAGE (MOBILE-FIRST) ───────────────────────────── Route: / (public) A. Navbar - Logo left, nav links center (Browse, How it Works, About), auth buttons right - Mobile: hamburger menu → slide-down drawer - Sticky on scroll, add subtle shadow after scrolling 60px B. Hero Section - Full-width, min-height 90vh on desktop / 70vh on mobile - Background: soft green gradient mesh (CSS only, no image dependency) - Headline: "Swap Clothes. Save the Planet." (large, bold, animated fade-up on load) - Subheadline: "Give your wardrobe a second life — trade directly with people near you." - Two CTAs: "Start Swapping" (primary, green filled) + "Browse Listings" (secondary, outlined) - Floating item card preview (absolutely positioned, CSS animation: gentle float up/down) C. How It Works — 3 Steps - Section title: "How SwapStyle Works" - Three cards in a row (stack vertically on mobile): Step 1 — List: "Upload photos and describe what you want to swap" icon: ti-camera Step 2 — Browse: "Find items you love from people near you" icon: ti-search Step 3 — Swap: "Send a swap request and exchange directly" icon: ti-arrows-exchange - Animated connecting dashed line between steps (desktop only, CSS) - Each step card: step number circle, icon, bold title, short description D. Featured Listings Grid - Section title: "Fresh Listings Near You" - Fetch: GET /api/items?limit=8&sort=newest (call on mount) - Responsive grid: 1 col (mobile) → 2 col (sm) → 4 col (lg) - Each card: image (aspect-ratio: 3/4, object-cover), title, condition badge, owner chip (avatar + name), "View Item" button - Skeleton loading state (pulse animation) while fetching - "See All Listings" button below grid → navigates to /browse E. Sustainability Counter - Full-width green-tinted banner section - Fetch: GET /api/stats/sustainability → { itemsSaved, co2Saved, waterSaved } - Three animated counters (count up from 0 on scroll into view using IntersectionObserver): 🌱 X items saved from landfill 💧 X litres of water conserved 🌍 X kg of CO₂ avoided - Short copy below: "Every swap is a small act of climate action." F. Testimonials Strip - Section title: "What Swappers Say" - Fetch: GET /api/reviews?featured=true&limit=4 - Horizontal scroll snap on mobile, 4-column grid on desktop - Each card: quote text, star rating, user avatar + name + city - Fallback: hardcode 3 placeholder testimonials if API returns empty G. Footer - Logo + tagline left - Links: About, FAQ, Contact, Privacy Policy - Social icons: Instagram, Twitter/X, Facebook (ti icons) - "© 2025 SwapStyle. Wear it forward." tagline ─── BACKEND ROUTES NEEDED ─────────────────────────────────────── Add these if not already present: GET /api/users/me/stats → { totalListed, successfulSwaps, trustScore } GET /api/users/me/saved → array of saved items (populated) DELETE /api/users/me/saved/:id → remove from saved list GET /api/swaps?type=incoming → filter by receiver = currentUser GET /api/swaps?type=outgoing → filter by requester = currentUser GET /api/swaps?status=completed → filter completed swaps for current user GET /api/items?owner=me → items belonging to current user GET /api/stats/sustainability → aggregate counts from swap collection GET /api/reviews?featured=true → top-rated reviews, limit param supported POST /api/reviews → { swapId, rating, comment } ─── DELIVERABLES ───────────────────────────────────────────────── Provide complete code for: Dashboard page: pages/Dashboard.jsx (tab routing, stat cards, tab container) components/dashboard/StatCards.jsx components/dashboard/MyListings.jsx components/dashboard/SwapRequests.jsx components/dashboard/SavedItems.jsx components/dashboard/SwapHistory.jsx components/dashboard/ReviewForm.jsx (inline star rating + comment) Homepage: pages/Home.jsx (assembles all sections) components/home/HeroSection.jsx components/home/HowItWorks.jsx components/home/FeaturedListings.jsx components/home/SustainabilityCounter.jsx components/home/Testimonials.jsx components/home/Footer.jsx components/shared/ItemCard.jsx (reusable card used in grid + saved items) components/shared/StatusChip.jsx (reusable: Pending/Active/Completed etc.) Backend additions: routes/statsRoutes.js controllers/statsController.js additions to userRoutes.js and swapRoutes.js per routes listed above Follow these constraints: - All Tailwind: no custom CSS files; use Tailwind utility classes throughout - Mobile-first breakpoints: default = mobile, sm: tablet, lg: desktop - Use axios with the existing api.js interceptor (attaches JWT automatically) - Use React Query (useQuery / useMutation) for all data fetching and cache invalidation - Use React Router v6 (useNavigate, useSearchParams for tab persistence) - Skeleton loaders on every data-fetched section (no blank flashes) - All destructive actions (delete listing, decline swap) use a confirmation modal - Reuse existing auth context (useAuth hook) to get currentUser - Optimistic UI updates for: unsave item, accept/decline swap status chips
